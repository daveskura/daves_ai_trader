

Design / Logic Issues
8. Strategy 14 (passive) can never grow beyond 2 positions
run_passive hard-codes top2 = [...[:2]] and breaks after 2 holdings. If the goal is to benchmark against the S&P 500, holding only 2 stocks (even if they're the largest cap) produces highly variable returns that aren't representative of the index. Consider holding 5–10 positions or using SPY directly.
9. Stop-loss and minimum hold period interact poorly for PEAD (S07)
Strategy 07 depends on abnormal_return as a key signal. The abnormal_return in the KPI file is computed at morning run time and goes stale. update_quotes.py refreshes it, but only if it's run separately after market close. If strategy_runner.py is run before update_quotes.py, S07 is making decisions on 8+ hour old signals. You already noted this in the comments, but there's no enforcement or warning in strategy_runner.py itself to flag this.
10. leaderboard_history.csv deduplication only removes today's entries
pythonexisting_history = [r for r in read_csv(hist_path) if r.get("date") != today]
This correctly avoids duplicate dates, but if the file grows unbounded it'll eventually be very large. There's no archiving or max-rows logic.
11. score_dividend_growth fallback branch ignores dividend entirely
When dy == 0, the function falls back to scoring by beta and P/E with no dividend component at all. This means non-dividend payers can score highly in a dividend growth strategy, which defeats the strategy's purpose. These stocks should simply be excluded.
12. Commission model understates real costs for small positions
At STARTING_CASH = $1000 with COMMISSION = $1.00, a single buy+sell costs $2 minimum on a ~$500–600 position (0.3–0.4%). This is actually reasonable for modern brokers, but the SPREAD_PCT = 0.0005 (0.05%) is very optimistic for small-cap or less liquid names in your universe.
13. Claude prompt context window pressure
The ask_claude prompt includes full holdings detail and 10 candidates. As trades accumulate and position counts grow, the prompt stays bounded (max 3 positions, top 10 candidates), which is good. However, the reason strings in candidates can be long — worth truncating them explicitly rather than relying on Claude to ignore excess detail.

Minor Issues
14. _trading_days_held doesn't account for market holidays
It counts weekdays only, not actual trading days. A position bought before a long weekend (e.g. Thanksgiving) will appear to have been held longer than it actually was in market terms. For a paper trader this is low stakes, but it could cause premature sell signals for strategies with short hold periods like S02 (3 days).
15. reset_accounts.py uses a different STRATEGIES list than strategy_runner.py
reset_accounts.py has 15 entries; strategy_runner.py also has 15, but they're defined independently. If you add a strategy to one file, you have to remember to add it to the other. Consider importing STRATEGIES from a shared constants file.
16. equity_kpi_analyzer.py five_year_avg_dividend_yield normalization comment is correct but fragile
You correctly normalize the yfinance quirk (returns as % not decimal). Add an assertion or log warning if the raw value seems already normalized (e.g. raw_dy5 < 0.20) to catch if Yahoo ever fixes this inconsistency.
